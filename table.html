Table

Easily organize data in tables with nice looking views and styles.

Introduction

Tables are a nice way to organize a lot of data in rows and columns. We provide a few utility classes to help you style your table as easy as possible.

Browsers provide a default syle for table. We dont tamper with the browser's default style. Rather we create a class to diffentiate our table default from the browser's table default. Add the table class to a table to see how our typical table looks like. See example below.

Cell Width Fixer
The width of a column in a typical table cannot be absolutely determined. The table will consider the table's width, the number of columns in the table and the content of each cell to determine the width of each column in the table (ie. column that contain a cell with more content will have more width). But a column automatically set to have it min-width to be equals to the width of the longest word in all the cells in that column. So if you give a cell a size that is more than the size the table can provide, the cell size will be equal to the available size provided by the table. Even if the table is given a width:auto, the table prefers to set its max-width to its parent width.

We provide the w-fix class to be added to a table to increase the width of a cell to a desired size. Not that you will not be able to decrease the size of a cell with this fix just that the table will determine the minimum size of each cell. See example below.

Note that the total width of a cell given a size with the fix provided will be equal to given width + cell padding-left + cell padding-right.
Note that the size given to a cell affect its column. The w-fix class set the table width to auto. But if the table is given a specified width and the sum of width given to all columns in the table is not up to 100% width of the table, spaces will be added automatically to those columns.

Scroll Table

If your table happens to be wider than its container or would eventually get too wide on a specific screen size maybe due to the many columns used or cell size specified, don't worry. We already provide a fix for that. Just wrap the table inside a div element and add the table-scroll class to the div. This creates a container that provides a horizontal scrollbar whenever the table inside it are wider than the container itself.

Repsonsive Table

You can also make your table responsive to a considerable viewport or screen size. We provide the responsive class to be used on table that may deform the page flow on a tablet screen and below or on a screen of 720px maximum and below. See example of how it works below. Try viewing this page on a screen not more than 720px width to see the responsiveness.

Note that the responsive class will rotate the table to the left by 90deg (without affecting it contents) so that the rows will display as columns and the table body will be horizontally scrollable if the screen size does not contain it.
Also note that the responsive class will might affect the table style. So if you did not like the way it views your table, you can use the scroll table rather.
The tfoot will not be displayed at all when table is responsive. We expect to have the same content in the thead as in tfoot.
