The input-box and its components

The input element can be extended with addons i.e button, label, select, ..., and icon. These is done by creating a div that has an input-box class and adding a box-like form component (like input[type=text], select, etc.) that has the subject class, and any desired components to the div.
See different ways to use it below.

Icon Input-box

A box-like form component can have an icon inside of it as a badge.


When using component like select, you have to be mindful of where you place your barge icon to avoid overlapping of icons.

Form Component Variation

There are variations in the way each form component displays. It could in type, state or size.

Types

Input, select and textarea elements can appear in three diffrent types. The layout type, the transparent type and the inverted type.

Layout

There are two types of layout that an input, a select and a textarea can appear. The box layout and the lined layout. The box layout is the default layout and it is created by the normal syntax of input, select and textarea. Whereas to create an input, a select or a textarea with the lined layout, you need to add the lined class to it.

Transparent

Inputs, selects and textareas can be transparent. To create a transparent input, select or textarea, just add the transparent class to the element.

Note that The content or the placeholder is the only visible area of a transparent input, select or textarea.


Inverted Input

Inputs, selects and textareas can be inverted. And this is done by adding the inverted class to the element. The inverted class gives the element a black background and a white text color.



States
Disabled

Inputs, selects and textarea can appear to be disabled. That is done by simply adding the disabled attribute or class.



Notification state

An Input, select and textarea can show a positive or negative state. This is done by adding the state name as class ie. positive and negative classes.





Sizes

We provide four (4) different classes that can be used to adjust the font size of inputs, selects and textareas. They are mini, small, big and huge. The input element will be used as an example.





Tweaks and Utilities

We provide you with tweaks and utilites to create a nice looking form components.

Text Fitted Components

Inputs, selects and textareas can be formated in a way to fit-in within text. This is done by adding the text-fit class to an element to format.



Colour Clue

A transparent input, select or textarea can be given a background that makes it appear in relative to it parent background. We provide two different colour clue classes. They are the light-clue and the dark-clue.






Field

A field is provided as a box to section form components in a form. It could be used as a container for a label and a form component and it also enables some styling possibilities ie. label-placeholder and prefixed field. It is a div element with a field class.

A field can be formated to display a label alongside the principal form component by adding the inline class to the field.
A Label Placeholder Field

A label placeholder field is created by adding the label-placholder class to a field that contain an input with the required attribute and a label in respect order ie. the input should be placed before the label. It appears more fitted with a lined input.

Note that if the input used has a placeholder attribute, the Label will remain has a label
A prefixed field

A field can be prefixed with an icon. This is created by adding the prefixed class to a field and an icon placed before the main form components in the field.

Fields

Fields is used as a container for two or more field. It can be used to equally size the fields in it or used as a flex container that enables individual fields in it to be given different sizes and still remain inline. We provide 11 classes to equally size fields and 10 classes to individually size fields in a fields component.

To equally size fields in a fields container, a number from 2 to 12 is added as a prefix to the fields classed (seperated with hypen) of a div used to wrap two or more field. ie. <div class="x-fields">...</div> where x is a number from 2 to 12.

To individually size a field and retain it inline status, the fields container in is also required. In this case, it is used as a flex container while the individual field is given size from the grid class. The -wide class that is prefixed by a number from 2 to 12 is given to the field ie. <div class="fields"><div class="field x-width">...</div>...</div> where x is a number from 2 to 12.

  • A single label can be used with multiple fields by using the fields container as the main form component in a field.
  • Fields container is responsive. It makes all the fields in it fluid at a medium screen and down
