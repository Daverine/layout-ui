Menu

Defines different styles and layout for a menu.

Introduction

A menu use to display items for navigation actions. All versions of a menu are a menu classed element filled with item classed elements.

You can add the active class to any item to denote an active item.

Fillable Menu and Adaptable Items

Items can also be set to expand out and take up an even amount of space, with the fillable class added to its menu. The items will automatically size themselves equally inside the menu. Using the adaptable class on an item or items will make them take up the available space in the menu.

Bordered Menu

A menu can have border around it. Just add the bordered class to the menu.

The first and last item of a bordered menu depend on its parent left and right border respectively. The bordered class can be added to the item to give it a left or a right border.

Logo

A menu can contain logos. Insert the logo in a menu item and give a class logo.

A logo in a menu is given max-height: 2.5em. ie. 2.5 of the menu's font-size.

Stackable Menu

A menu can stack at mobile resolutions. Just add the stackable class to the menu.

Stackable menus are intended to be used with only simple and bordered menu content. Stacked menus will not replicate all additional stylings for vertical menus like adjusting dropdown position.

Menu Size

A menu can vary in size. We already provide some predefined sizes to size a menu. The example of those sizes is shown below with the class needed on them.

  • All component of a menu are sized relative to the menu's font size. So when the font size of the menu is changed, it affect the whole size of the menu and it components.
  • The default font size is 1rem. You can make the menu font size to be directly relative to it parent (ie. em instead of rem font unit.) by giving it the emd class.
